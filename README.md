# 美股／台股 AI＋太空 成長股／ETF 價格帶監測器

把財報達人 **孫慶龍《AI 護國群山投資藍圖》** 的估價方法論，加上你 Computex／NVIDIA ITCC 的觀察，做成一支可即時跑的監測工具。涵蓋 **台股（FinMind）＋ 美股（yfinance）**，自動把現價分類到 **大特價／便宜／合理／昂貴／瘋狂** 五個價格帶，在進入便宜價時提醒，並用你的投入資金做情境式投報率試算。

> ⚠️ **這是資訊／教育工具，不是投資建議。** 所有「便宜價、目標價、報酬率」都建立在 `watchlist.yaml` 裡**你可以修改的假設**之上。免費數據為**日收盤價(EOD)**,非盤中即時。投資前請自行查證並自負風險。

- 所有報價皆為**日收盤價（EOD）**,非盤中即時(見第 5 節盲點④)。

## 4. 估價方法論（孫慶龍法的程式化）

**核心**：成長股不適合用「殖利率法」算便宜價（它們幾乎不配息）。藍圖改用**前瞻本益比河流圖**：

```
未來 EPS = 基期營收 × (1+營收CAGR)^年數 × 稅後淨利率 ÷ 股數
五個價格帶 = 未來 EPS × 各本益比倍數（大特/便宜/合理/昂貴/瘋狂）
```

> 📌 **本清單已校正（2026/6）**：除 PDF 四檔（台積電保留原推導鏈）外，所有 `forward_eps`／`forward_revenue` 已用 **yfinance 分析師共識 + 多代理對抗式驗證** 校正，五價格帶由各檔 `fair_multiple` 依河流圖幾何展開。原始 PDF 種子版備份在 `watchlist.pdf_seed.yaml`；要重新校正見 `_gen_watchlist.py`＋`_calibrations.json`。校正後浮現買點：**NVDA／MU／LUNR 落在便宜價區**（NVDA forward P/E 僅 16x，印證「forward P/E 仍偏低」）。

`watchlist.yaml` 支援三種估價法：

- **`pe_band`**：會賺錢的成長股。`forward_eps`（或 `derive` 推導鏈）× `pe_bands`。
  - `pe_bands` 可填 `auto` → **台股**用 FinMind 真實歷史本益比百分位（河流圖）。
  - ⚠ **美股不要用 auto**：yfinance 無乾淨歷史本益比，近似法對「EPS 暴增股」會失真（NVDA 便宜帶會算出 PE≈3 的荒謬值），故美股一律用**明確前瞻本益比帶**。
- **`ps_band`**：燒錢中的成長股（多數太空股）。`(forward_revenue / shares) × ps_bands`，本益比無意義時改用股價營收比。
- **`price_band`**：最保守後備（市值型/主題型 ETF、難估個股）。直接用過去 N 年股價分布百分位，**完全不靠 EPS 假設**。
- **`yield_band`**（殖利率河流圖）：**高股息 ETF**。`年化配息 ÷ 殖利率帶`，**殖利率越高＝越便宜**；`yield_bands: auto` 用歷史殖利率百分位自動推。

> 🧩 **台股 ETF 已納入**：市值型（0050、006208）、半導體主題（00891）用 `price_band`；高股息（0056、00878、00919、00929、00940）用 `yield_band`。
> 🔧 **台股價自動還原分割**：FinMind 原始價未還原,程式只在「隔日跳動屬整數倍拆股（r<0.6 或 r>1.7）且非現金除息日」時才回溯還原（例：0050 在 2025/6 做 1:4 分割），避免把高股息 ETF 的大額除息誤判成分割，修正百分位／波動率／殖利率河流圖。

## 5. 你的規格盲點與我如何處理（需求 ④ 的正式回答）

> 你要我「判斷規格是否有盲點或 bug」。以下是我認為最關鍵的幾點——有些已內建到工具裡。

1. **「便宜價」是判斷、不是事實（最大單點故障）。** 便宜價 = 未來EPS × 低本益比，而「未來 EPS」是人為假設；改一個 CAGR，便宜價可差 30%＋。
   → 處理：所有假設攤在 `watchlist.yaml`、可改可版本控；報表同時顯示「現價隱含本益比」並在**假設可能過時／過低時主動警告**（例如台達電 forward P/E≈92 遠高於你設的帶，提示 EPS 假設要校正）。**聯發科是最佳例證**：用保守 2027 EPS=132 顯示「超瘋狂」，但若用 PDF 樂觀 2028 EPS=400，同價位變「便宜」——分類完全取決於你錨定哪一年。

2. **「預測便宜價何時出現」在數學上不可預測。** 那是市場擇時，沒人能可靠做到。
   → 處理：**不**給假裝精準的日期。改給「還要跌幾 %／歷史落點百分位／波動率觸及機率」三個可驗證的量，並明確標示是統計估計而非預言。

3. **「投報率預估」同理。** 未來報酬不可知。
   → 處理：改做 **if-then 情境報酬**（若回到各價格帶，1/3/5 年報酬與年化），明白標示是模型不是保證。

4. **「即時」其實是日收盤價(EOD)。** 本工具抓的是**日線收盤**,不是盤中即時:台股 FinMind 盤中顯示前一交易日收盤、收盤後(約傍晚)才更新當日收盤;美股 Yahoo 盤中那根約延遲 15 分、收盤後為正式收盤。對長線價格帶判斷夠用;要秒級即時需券商 API 或付費 feed。

5. **稅費與匯率不能忽略。** `roi` 已內含台股證交稅 0.3%＋手續費、美股外國人股利預扣 30%、台幣/美元即時匯率與匯率風險提示。

6. **集中度風險。** 這份清單幾乎全是 AI 半導體，高度相關、會一起漲一起崩。請自行控管部位，勿視為分散投資。

7. **選股的追高風險。** 「用近期資訊找成長股」若變成追熱度，會在瘋狂價買進。故清單用**透明主題條件**而非黑箱「AI 選股」，且 `screen` 只回答「誰目前最便宜」，避免追高。

8. **數據可靠性。** yfinance 是非官方介面、機房 IP 易被限流；FinMind 免費 300 次/小時(設 `FINMIND_TOKEN` 可到 600)。工具對抓取失敗會優雅降級(重試＋過期快取保命)、給清楚訊息，而非整支崩掉。

9. **太空股估值。** ASTS/LUNR 等仍虧損，本益比無意義 → 已改用 `ps_band`（股價營收比）。

10. **這是工具不是建議。** 「提醒買進」＝「價格進入你預先定義的便宜區」，不是「你該買」。

## 6. 我額外加的實用功能（需求 ⑤）

- **本益比河流圖 auto 模式**（台股）：用真實歷史 PER 百分位自動推價格帶，不必手填。
- **假設健全度護欄**：現價隱含本益比 vs 你設定的帶，過時即警告。
- **波動率觸及機率**：用幾何布朗運動首次穿越公式，誠實量化「便宜價多快可能出現」。
- **三種估價法自動適配**：賺錢股→PE、燒錢股→PS、難估者→純股價分布。
- **跨幣別投報率**：台幣投入買美股，自動換匯並提示匯率風險。
- **報價快取**：避免短時間重複打 API。
- **桌面通知 + 常駐監測**：`watch` 在進便宜價時主動通知。
